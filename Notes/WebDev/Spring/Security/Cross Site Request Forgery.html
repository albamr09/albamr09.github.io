Cross Site Request Forgery

Spring Security protects against Cross-Site Request Forgery. CSRF is a security attack where a website tricks you into executing an action on a web application that you are currently logged in. Protection from this type of attack is embedded in the Spring Security Filters.

This protection is enabled by default. Spring Security uses the Synchronizer Token Pattern, where each request includes a session cookie and a randomly generated token. So for request processing, Spring Security verifies the token before processing.

How to use it?

  • For form submissions use "POST" instead of "GET"
  • The Spring Security tag <form:form> automatically adds the CSRF token.
  • If you do not use the tag, you must manually add the CSRF token.
  • If you do not add the token you get an error message: 403 Forbidden, and further information about how the token cannot be null.
